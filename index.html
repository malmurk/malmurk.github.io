Flagship case study

Multilingual product classifier

Problem

E-commerce catalogs arrive as messy, multilingual product lists. Sorting them by hand into a taxonomy of 3,000+ categories is slow and error-prone, and it doesn't scale past a few thousand SKUs a day.

Approach — a three-stage cascade

Instead of picking one class out of 3,000 in a single shot, each stage narrows the decision and structurally masks the next stage's invalid categories at inference — the parent model can't propose a leaf that doesn't belong to the super-category it already chose.

Super

~20 categories

Parent

~200 categories

Leaf

3,000+ categories

Each stage is an XLM-RoBERTa classifier fine-tuned for its position in the cascade.

In production

  • INT8-quantized, served with ONNX Runtime
  • Predictions at or above 0.93 confidence auto-assign; everything below drops into a human-review queue
  • Trained on a consumer AMD GPU via ROCm
